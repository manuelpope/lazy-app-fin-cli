FinLens

Terminal UI para señales de trading: indicadores técnicos, CAPM, GJR-GARCH, métricas de riesgo y análisis LLM vía Ollama. Datos frescos de Yahoo Finance.

$ uv run finlens       # TUI interactiva (con uv)
$ finlens              # o si está instalado
$ finlens scan         # escaneo headless
$ finlens --help       # ayuda CLI

Instalación

Requisitos

  • Python 3.12+
  • Ollama (opcional, para análisis LLM)
  • Modelo recomendado: qwen3.5:9b-mlx o phi4

Instalar

git clone <repo> && cd finlens

# Opción A — uv (recomendado)
uv sync
uv run finlens

# Opción B — pip
pip install -e .
finlens

# Modelo LLM opcional
ollama pull qwen3.5:9b-mlx

Se carga automáticamente al iniciar la app vía python-dotenv.

Mercados preconfigurados: US (^GSPC benchmark), Argentina BYMA/CEDEARs (^MERV benchmark), Emerging Bonds (EMB self-benchmarked). Risk-free rate: 4.5% US, 40% Argentina, 5% Bonds.

Uso — Terminal UI

Ejecutar finlens. La pantalla se divide en tres áreas:

  • Sidebar izquierdo — árbol de mercados y leyenda de teclas
  • Tabla central — matriz de señales (ticker, precio, score compuesto, señal)
  • Panel derecho — detalle del ticker seleccionado (indicadores, CAPM, GARCH, riesgo)
  • Footer — barra de estado con bindings activos

Señales y Score Compuesto

Cada ticker se evalúa con 4 indicadores técnicos, más CAPM y métricas de riesgo. El score compuesto (-1.0 a +1.0) combina todo:

ComponentePesoQué mide
SMA crossover15%Cruce de media rápida (20d) sobre lenta (50d)
RSI20%Momento: sobrecompra (>70) / sobreventa (<30)
MACD15%Convergencia/divergencia de medias móviles
Bollinger Bands15%Anchura de bandas y posición del precio
CAPM alpha15%Alpha anualizado (regresión t-dist)
Risk ratio20%Sharpe + Sortino + Calmar combinados

Señal final:

  • BUY — score ≥ +0.30
  • HOLD — score entre -0.30 y +0.30
  • SELL — score ≤ -0.30

Modelos Analíticos

CAPM Heavy-Tailed

Regresión lineal de retornos del activo contra el benchmark del mercado usando distribución t de Student (en vez de normal). Esto captura mejor las colas pesadas típicas de mercados emergentes. Se estima por máxima verosimilitud (scipy.optimize.minimize).

  • alpha — retorno anómalo (anualizado)
  • beta — sensibilidad al mercado
  • R² — bondad de ajuste
  • p_value_alpha — significancia estadística del alpha

GJR-GARCH(1,1)

Modelo de volatilidad asimétrica que captura el efecto leverage: las malas noticias incrementan más la volatilidad que las buenas. Estimado con arch_model usando distribución t.

  • omega, alpha, gamma, beta — parámetros del modelo
  • persistence — persistencia de shocks de volatilidad (alpha + gamma/2 + beta)
  • annualized_vol — volatilidad anualizada proyectada

Análisis LLM (Ollama)

El análisis LLM envía los datos del ticker (señales, CAPM, GARCH, riesgo) a un modelo local vía la API compatible con OpenAI de Ollama. El modelo devuelve un análisis narrativo con recomendación y confianza.

Rendimiento: Modelos "thinking" como qwen3.5 pueden tardar 30-50s por ticker porque generan razonamiento interno extenso antes de la respuesta. Para más velocidad, usar modelos como phi4 o granite4.1:3b.

Formato del prompt

El prompt incluye: ticker, precio, cambio %, señales individuales (SMA/RSI/MACD/BB), CAPM (β, α, R²), GARCH (σ, persistencia), métricas de riesgo (Sharpe, Sortino, MaxDD), y score compuesto.

Formato de respuesta esperado

El modelo debe responder con tres marcadores:

[NARRATIVE] El activo muestra momentum alcista con indicadores alineados...
[CONFIDENCE] 0.72
[RECOMMENDATION] BUY

Si faltan marcadores, se usan defaults (HOLD, confianza 0.5, narrative truncado a 400 caracteres).

Los resultados aparecen en un overlay modal. Presionar Esc para cerrar.

Cache de Datos

Los datos descargados de Yahoo Finance se guardan localmente en .finlens_cache/ como archivos Parquet. Cada ticker tiene su propio archivo. La próxima vez que se ejecute la app, si el cache tiene menos de max_age_hours (default 24), se reusa. Para forzar refetch, usar s o finlens scan --force.

Modo Headless (CLI)

Ejecutar escaneo sin interfaz gráfica. Útil para automatización o scripts:

finlens scan                         # escaneo normal con cache
finlens scan --force                 # forzar descarga fresca
finlens scan --verbose               # más output
finlens scan --config /ruta/config.yaml

El modo headless imprime una tabla resumen en consola con todos los tickers, precio, cambio %, señales y score compuesto.

Troubleshooting

No se ven datos / "Loading..." eterno

Verificar conexión a internet. Yahoo Finance puede bloquear requests masivos. Reintentar con s o finlens scan --force --verbose para ver errores.

El overlay LLM no aparece

Verificar: (1) Ollama corriendo (ollama serve), (2) modelo descargado (ollama list), (3) llm.enabled: true en config.yaml, (4) puerto 11434 accesible.

"No ticker selected" al presionar l

Navegar con flechas a una fila de la tabla primero, o presionar l de nuevo (usa el primer ticker filtrado como fallback).

Errores de conexión Ollama

Asegurar que Ollama está corriendo: curl http://localhost:11434/api/tags. Si usás un host/puerto distinto, actualizar llm.api_url en config.yaml.

Cache corrupto

Eliminar la carpeta .finlens_cache/ manualmente: rm -rf .finlens_cache.
